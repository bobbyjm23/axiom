Governance · July 2026

From shadow AI to approved assistant

How managing partners and GMs can replace unsanctioned ChatGPT with a tool people actually want to use — in one quarter.

Blocking ChatGPT doesn't work. Your people have seen what AI can do for drafting, summarisation, and research — and they'll find a way. The answer isn't prohibition; it's substitution with something better that happens to be compliant.

Enterprise governance platforms like Cetus AI's Songlines Control excel at discovering shadow AI, attributing token costs, and enforcing runtime policy. That's critical at ASX scale. For a 80-person law firm or advisory practice, the first problem is simpler: give staff an approved assistant with the same UX as ChatGPT, running on infrastructure you control.

"AI doesn't behave like software licences. It behaves like a meter running 24/7 — and right now, no one is watching the meter."

— Cetus AI (competitor positioning on shadow AI risk)

The substitution playbook

We've run this pattern across regulated mid-market buyers. Four phases, 10–14 weeks:

Phase 1: Acknowledge and classify

Run a 2-week discovery workshop. Map which teams use shadow AI, for what tasks, and with what data sensitivity. Classify documents: public, internal, confidential, legally privileged. This isn't surveillance — it's understanding substitution requirements.

Phase 2: Pilot with champions

Deploy Sovereign Warden to 15–30 users in one practice group. Pre-load RAG workspaces with templates, policies, and precedents they'll actually query. Distribute the desktop app — the ChatGPT-class interface drives adoption, not another API portal.

Phase 3: Measure what matters

Track weekly active users, queries per user, time saved on drafting tasks, and qualitative feedback. Compare against baseline shadow usage (honest self-reporting in pilot groups works better than network scanning for firms under 250 people).

Phase 4: Policy + production

Update acceptable use policy to name Sovereign Warden as the approved tool. Block is the wrong word — redirect is better. Communicate that the approved tool is faster for firm-specific work because of RAG over internal documents.

UX is the governance strategy. Employees rejected API-only enterprise AI for a reason. AnythingLLM's desktop interface is the adoption moat — same chat patterns as ChatGPT, without the compliance hangover.

What not to do

  • Don't run a 6-month RFP for custom ML — you'll lose the organisation to shadow tools while you procure
  • Don't pilot with an API and no desktop app — adoption will fail
  • Don't skip executive sponsorship — one managing partner email changes behaviour
  • Don't treat the pilot as a throwaway — Sovereign Warden's production stack is the pilot stack
