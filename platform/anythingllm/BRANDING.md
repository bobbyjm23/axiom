# AnythingLLM Branding Guide (POC)

Branding is applied through AnythingLLM’s **native Appearance settings**
(`custom_app_name` + uploaded logo in `storage/assets`), not via frontend
CSS/JS login overlays.

## What sticks (durable)

| Setting | Value | Mechanism |
|--------|--------|-----------|
| App name | `Sovereign Warden` | `system_settings.custom_app_name` |
| Logo (sidebar + login) | Full wordmark | `system_settings.logo_filename` → `storage/assets/sovereign-warden-logo.png` |
| Login copy | “Welcome to Sovereign Warden” / “Sign in to your Sovereign Warden account.” | Generated by AnythingLLM from app name |

### Docker image seed

On each build, `platform/anythingllm/branding/` is copied to
`/app/server/public/warden-branding/`. At container boot,
`ensure-branding.js` (hooked from `patches/apply-warden-patches.sh`) upserts:

- App name → `Sovereign Warden`
- Logo → wordmark PNG (`logo-light.png` for the default dark UI)

Set `WARDEN_BRANDING_FORCE=false` to leave a non-default logo the admin already
uploaded.

### Admin UI (manual / override)

**Admin → Appearance → Branding**

- App name: `Sovereign Warden`
- Upload logo: `platform/anythingllm/branding/logo-light.png` (dark UI) or
  `logo-dark.png` (light UI)
- Footer text (optional): `Powered by Sovereign Warden Platform — data stays in-house`

### Desktop / source builds

Default login SVGs in the fork:

- `desktop/anything-llm/frontend/src/media/illustrations/login-logo.svg`
- `…/login-logo-light.svg`

Synced from `platform/anythingllm/branding/login-logo*.svg`. Sign-in headings
stay stock AnythingLLM (`Welcome to {customAppName}`).

## Assets

```
platform/anythingllm/branding/
  logo-light.svg|png   # light wordmark (dark backgrounds)
  logo-dark.svg|png    # dark wordmark (light backgrounds)
  login-logo.svg       # default-theme login SVG
  login-logo-light.svg # light-theme login SVG
  ensure-branding.js   # boot seed into system_settings
```

## Electron layer

Separate from in-app branding — `desktop/config.json`:

```json
{
  "appName": "Sovereign Warden"
}
```

## Verify

1. Open `http://localhost:3000` (nginx → AnythingLLM).
2. Login page: stock split layout; wordmark logo; app name from settings.
3. No `/warden-branding/login-branding.css` or `.js` in page source.
4. After restart, app name + logo remain (volume-backed `system_settings`).
